# Code

> Shinyblocks function: `block_code()`
> Shadcn reference: documentation code examples on
> <https://ui.shadcn.com/docs/components/textarea>
>
> Per [ADR 0015](../decisions/0015-component-specs.md), every exported
> `block_*()` has a spec at this shape. Keep it tight — five sections,
> ~30 lines. Verbosity defeats the purpose.

## States

The visual states this component must render correctly. Quality Gate
item 15 walks each one against the showcase.

- **default** — Bordered, rounded, background-surface frame with
  transparent preformatted code content.
- **line numbers** — Left gutter numbers generated by CSS counters, with
  each code line aligned to the shadcn docs examples.
- **absolute copy button** — Quiet icon button positioned in the top-right
  corner of the code frame.
- **copied feedback** — Icon changes from standard copy to checkmark SVG with active color transition.
- **header (optional)** — Optional upper bar containing editor dots on the
  left, uppercase language label, and inline copy button.

## Token contract

Map visual roles to the semantic CSS variables that drive them. If a
role uses a hard-coded value rather than a token, that's a divergence
and goes in the next section.

| Visual role | Token |
| --- | --- |
| Container BG (default) | `var(--muted)` |
| Container BG (outline) | `transparent` |
| Border | `var(--border)` |
| Absolute Copy Button BG | `transparent` |
| Absolute Copy Button Hover BG | `var(--accent)` |
| Absolute Copy Button Hover text | `var(--accent-foreground)` |
| Line number text | `var(--muted-foreground)` |
| Text foreground | `var(--foreground)` |
| Language Label | `var(--muted-foreground)` |

## Deliberate divergences from shadcn

Anywhere shinyblocks knowingly differs from the shadcn reference, with
reasoning. Empty list = "matches shadcn".

- **Lightweight highlighter** — shadcn's docs examples are produced by the
  documentation pipeline (`rehype-pretty-code` / Shiki). `block_code()`
  preserves the surface, line gutter, copy action, and whitespace, with a
  small built-in tokenizer for common R, Python, and TSX examples rather than
  bundling Shiki.
- **Optional header extension** — `header = TRUE` adds an editor-style bar
  for dashboard use cases. It is not part of the default shadcn docs
  example.
