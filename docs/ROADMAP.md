# Roadmap

## Phase 0: Planning

- Define package scope.
- Decide asset strategy.
- Decide theme strategy.
- Define first component contracts.
- Write a minimal example target.

## Phase 1: Dashboard Shell

- `shadcn_page()`
- `shadcn_sidebar()`
- `shadcn_header()`
- `shadcn_nav_item()`
- CSS variables and base layout.
- Smoke tests and one example app.

## Phase 2: Core Components

- Cards and value boxes.
- Buttons.
- Badges.
- Tabs.
- Alerts.
- Dropdown/menu pattern if feasible without heavy JavaScript.

## Phase 3: Shiny Integration

- Input wrappers where useful.
- Output containers.
- Loading and error states.
- Responsive layout helpers.

## Phase 4: Documentation

- pkgdown site.
- Vignettes.
- Component gallery.
- Migration notes from `shinydashboard`.

## Phase 5: Hardening

- Accessibility pass.
- Cross-browser review.
- R CMD check.
- CRAN-readiness review if desired.
